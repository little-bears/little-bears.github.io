React 是用于构建用户界面的 JavaScript 库，这里主要简单介绍 JSX 语法与类组件的生命周期、初始化项目~

1、React特点

  React 起源于 Facebook 的内部项目，具有声明式编程、组件化、高效灵活等特点（JSX语法特点），这里主要介绍一下 JSX 这种 JavaScript 语法的扩展
  React 认为渲染逻辑本质上与其他 UI 逻辑内在耦合，并没有采用将标记与逻辑进行分离到不同文件的这种人为分离形式，而是通过将二者共同存放在称之为“组件”的松散耦合单元之中，来实现关注点分离，JSX 文件拓展名为’jsx’

  [注] 为了便于阅读，我们可以将 JSX 拆分为多行，将内容用括号 () 包裹，避免编译器在行尾自动添加分号找出编译错误
  形如上面的标签语法，在 JavaScript 中它既不是标签也不是字符串，在 JSX 语法中，可以在标签上添加属性，在大括号内放置任何有效的 JavaScript 表达式，例如简单计算、函数调用、逻辑运算、三目运算表达式等
  [注] JSX 语法上更接近 JavaScript 而不是 HTML，所以 React DOM元素使用 camelCase（小驼峰命名）来定义属性的名称。特别的，在 JSX 中，class属性变成了 className，而 tabindex 则变为 tabIndex

2、组件的生命周期

  React 的组件可以定义为 class 或函数的形式。class 组件目前提供了更多的功能
  (1) 定义 class 组件，需要继承 React.Component

  [注] 在 React.Component 的子类中有个必须定义的 render() 函数

  (2) 定义组件最简单的方式就是编写 JavaScript 函数（纯函数），该函数接收参数并返回一个 React 元素

  [注] 函数组件的定义除了这里使用的字面量形式，也可以使用 function 关键字进行定义

  特别的，类组件继承于React.Component，可以在组件中使用 state(组件状态) 以及生命周期函数。而函数组件为纯函数，不能在函数组件中使用state以及生命周期函数。为了在函数组件中使用类似于组件状态以及生命周期的功能，在 React 16.8 的新增特性中推出了 Hook。可以在不编写 class 的情况下使用 state 以及其他的 React 特性，如生命周期函数等，详见文章：《React函数组件与HOOK》

  (3) 类组件的生命周期

  React生命周期图谱

  其中，constructor、render、componentDidMount、componentDidUpdate、componentWillUnmount为主要的常用的生命周期，常用生命周期图谱如下：

  ① constructor(props)
  构造函数仅用于两种情况，通过给 this.state 赋值对象来初始化内部 state、为事件处理函数绑定实例

  [注] 只能在构造函数中直接为 this.state 赋值。如需在其他方法中赋值，你应使用 this.setState() 替代。要避免在构造函数中引入任何副作用或订阅。避免将 props 的值复制给 state，当 props 更新时 state 不会被更新

  ② render()
  render 被调用时，它会检查 this.props 和 this.state 的变化并返回以下类型之一:
  1) React 元素
  2) 数组 或 fragments
  [注] Fragments允许一个组件返回多个元素，允许你将子列表分组，而无需向 DOM 添加额外节点，使用<React.Fragment> 或 <> 标签来包裹多个元素，后者为简写形式，不支持 key 或属性
  3) Portals
  [注] Portal 提供了一种将子节点渲染到存在于父组件以外的 DOM 节点的方案
  4) 字符串或数值类型 (在 DOM 中会被渲染为文本节点)
  5) 布尔类型或 null (什么都不渲染，支持返回 boolean && <Child />的场景)

  ③ componentDidMount()
  该会在组件挂载后（插入 DOM 树中）立即调用，可以在这里获取DOM元素进行相应初始化等操作，一般在这个生命周期中通过网络请求获取数据、添加订阅

  ④ componentDidUpdate(prevProps, prevState, snapshot)
  在更新后会被立即调用，首次渲染不会执行此方法
  [注] 避免在 componentDidUpdate() 中直接调用 setState()，必须被包裹在一个条件语句里，否则会导致代码死循环。第三个参数 snapshot 为 getSnapshotBeforeUpdate() 生命周期函数返回的快照信息，未定义该函数时 snapshot 的值为 undefined

  ⑤ componentWillUnmount()
  组件卸载及销毁之前直接调用，一般在这个方法中清空计时器、网络请求与消息订阅等，不应当在此生命周期中使用 setState()，因为该组件将永远不会重新渲染

  其他不常用生命周期：
  ⑥ static getDerivedStateFromProps(props, state)
  在调用 render 方法之前调用，并且在初始挂载及后续更新时都会被调用。它应返回一个对象来更新 state，如果返回 null 则不更新任何内容

  ⑦ shouldComponentUpdate(nextProps, nextState)
  当 props 或 state 发生变化时，shouldComponentUpdate() 会在渲染执行之前被调用。首次渲染或使用 forceUpdate() 时不会调用该方法。目前，此生命周期函数返回 false，则不会调用render、getSnapshotBeforeUpdate、componentDidUpdate等

  ⑧ getSnapshotBeforeUpdate(prevProps, prevState)
  最近一次渲染输出（提交到 DOM 节点）之前调用。它使得组件能在发生更改之前从 DOM 中捕获一些信息。此生命周期方法的任何返回值将作为参数传递给 componentDidUpdate()
