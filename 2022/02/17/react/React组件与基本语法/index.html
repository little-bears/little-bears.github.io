2、元素渲染

  元素是构成 React 应用的最小砖块，与浏览器的 DOM 元素不同，React 元素是创建开销极小的普通对象。React DOM (ReactDOM.render方法)会负责更新 DOM 来与 React 元素保持一致。React 元素是不可变对象，更新 UI 唯一的方式是创建一个全新的元素，并将其传入 ReactDOM.render 方法

4、生命周期（React主动调用）

  React 的组件可以定义为 Class 或函数的形式。Class组件（继承于React.Component基类）功能更完善，React.Component 的子类中有个必须定义的 render 函数。常见的生命周期函数：

(1) render

  render() 是 class 组件中唯一必须实现的方法，且函数应该为纯函数。如需与浏览器进行交互，请在 componentDidMount() 或其他生命周期方法中执行你的操作
  当 render 被调用时，它会检查 this.props 和 this.state 的变化并返回React 元素、数组或 fragments、Portals、字符串或数值类型或布尔类型或 null
  [注] 如果 shouldComponentUpdate() 返回 false，则不会调用 render()

(2) constructor

  如果不初始化 state 或不进行方法绑定，则不需要为 React 组件实现构造函数
  在 React 中，构造函数仅用于以下两种情况：
    通过给 this.state 赋值对象来初始化内部 state
    为事件处理函数绑定实例
  [注] 在为 React.Component 子类实现构造函数时，应在其他语句之前前调用 super(props)；构造函数中直接为 this.state 赋值，其他方法使用 this.setState() 替代；避免将 props 的值复制给 state；要避免在构造函数中引入任何副作用或订阅

(3) componentDidMount

  componentDidMount() 会在组件挂载后（插入 DOM 树中）立即调用。依赖于 DOM 节点的初始化应该放在这里。如需通过网络请求获取数据、添加订阅（函数中调用 setState() 将触发额外渲染，此渲染会发生在浏览器更新屏幕之前）

(4) componentDidUpdate

  componentDidUpdate() 会在更新后会被立即调用。首次渲染不会执行此方法
  [注] 函数中可直接调用 setState()或请求，但请注意它必须被包裹在一个条件语句里，否则会导致死循环

(5) componentWillUnmount

  componentWillUnmount() 会在组件卸载及销毁之前直接调用。在此方法中执行必要的清理操作，如清除计时器，取消网络请求、订阅等
  [注] 函数中不应调用 setState()，因为该组件将永远不会重新渲染（组件实例卸载后将不会再挂载）

(6) 补充：组件其他API

  主要补充 setState、forceUpdate 方法以及 Context 组件共享数据方案
  不同于上述生命周期方法（React 主动调用），setState() 和 forceUpdate() 这两个方法可以在组件中手动调用

① setState

  setState() 将对组件 state 的更改排入异步队列等待主线程空闲后执行，并通知 React 需要使用更新后的 state 重新渲染此组件及其子组件。这是用于更新用户界面以响应事件处理器和处理服务器数据的主要方式
  React 并不会保证 state 的变更会立即生效，所以在调用 setState() 后不能立即读取 this.state，需使用 componentDidUpdate 或者 setState 的回调函数来获取更新后的状态（推荐使用 componentDidUpdate），这两种方式都可以保证在应用更新后触发
  若需要基于之前的 state 来设置当前的 state，可以使用 updater 函数

  updater 函数中接收的 state 和 props 都保证为最新。updater 的返回值会与 state 进行浅合并

② forceUpdate

  默认情况下，当组件的 state 或 props 发生变化时，组件将重新渲染。若 render 方法还依赖于其他数据，则可以调用 forceUpdate() 强制让组件重新渲染（尽量避免使用）

6、表单

  受控组件与非受控组件：
  表单元素通常有自己的状态
  React可以设置 state 成为“唯一数据源”来管理表单元素。渲染表单的 React 组件还控制着用户输入过程中表单发生的操作。被 React 以这种方式控制取值的表单输入元素就叫做受控组件
  相对于受控组件，将表单元素状态交由本身 DOM 节点来维护，通过使用 ref 来从 DOM 节点中获取表单数据的元素称为非受控组件

  [注] Refs 是使用 React.createRef() 创建的，并通过 ref 属性附加到 React 元素。当 ref 属性用于 HTML 元素时，构造函数中使用 React.createRef() 创建的 ref 接收底层 DOM 元素作为其 current 属性。当 ref 属性用于自定义 class 组件时，ref 对象接收组件的挂载实例作为其 current 属性。不能在函数组件上使用 ref 属性，因为没有实例（可以函数组件中使用相应的 HOOK 来完成此功能）

7、状态提升

  在 React 应用中，任何可变数据应当只有一个相对应的唯一“数据源”。通常，state 都是首先添加到需要渲染数据的组件中去。如果其他组件也需要这个 state，可以将它提升至这些组件的最近共同父组件中，应当充分依靠自上而下的数据流
