Redux 是集中式存储和管理应用的状态的 JavaScript 应用的状态容器，状态变化可预测，也支持其他 UI 框架，React Redux 是 Redux 官方为 React UI 维护的状态容器，能使用组件无视层级关系进行通信（一般使用于较为复杂的大型项目中，一般项目 React 数据模型已满足需求）。Redux三大原则：单一数据源、State是只读属性、使用纯函数来修改状态。示例使用更推荐的函数组件形式编写~

3、创建状态以及更新逻辑并配置到 store

  ① 使用 createSlice （@reduxjs/toolkit 的功能）创建 Redux 状态管理逻辑片段（Slice）
  Slice 是 redux toolkit 定义的一个概念，是一系列相关数据的状态管理，包含 state，reducer，action creator（@reduxjs/toolkit 这样的文件结构使得配置 store 时，更加简洁）。使用 createSlice，可以基于 reducer 函数自动生成对应的 action creator 来返回对应的 action 对象
  这里定义一个计数器状态的更新逻辑，createSlice 接收一个配置参数的对象（名称标识，初始状态，以及一个或多个 reducer 函数等）来定义并描述如何更新状态。在文件中导出对应的 reducer、action creator 函数

  [注] 其中 increment reducer、decrement reducer 接收参数的方式不一致是因为 App.js 使用的 dispatch 方法稍微有区别，触发 increment reducer 是直接调用本文件暴露出去的 action creator 函数，会返回一个 action 对象并触发更新，触发状态更新是通过 dispatch 触发 type 为 ‘count/decrement’ 的 action 对象来实现的, 这里还定义了参数 value 为 1
  [注] @reduxjs/toolkit createSlice 包括了 @reduxjs/toolkit 中的 createReducer 和 createAction 方法
