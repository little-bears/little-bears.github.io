React函数组件的Hook

2022-06-14
React
字数统计: 2.4k   |   阅读时长≈ 10 分钟

React 的组件可以定义为 Class 或函数的形式。本篇主要介绍函数组件及对应的Hook
Hook 使你在非 Class 组件的情况下可以使用更多的 React 特性（props， state，context，refs 以及生命周期等）
Hook将组件中相互关联的部分拆分成更小的函数（比如设置订阅或请求数据），并非强制按照生命周期划分，使编码更加灵活，代码更易理解

常见 Hook

  Hook 就是 JavaScript 函数，可以通过它使用 React 特性。使用遵循如下 Hook 规则：
  ① 只能在函数最外层调用 Hook（使得 Hook 的调用顺序在每次渲染中相同）, 可以在单个组件中使用多个 State Hook 或 Effect Hook，React 会按照声明的顺序依次调用执行
  ② 只能在 React 的函数组件（或自定义Hook）中调用 Hook，在其他 JavaScript 函数、Class 组件内部不起作用

1、useState

  useState() 方法里面唯一的参数就是初始的状态，可以是数字、字符串、对象或函数（返回计算后的值，函数只在初始渲染时被调用），调用后返回当前状态已经更新此状态的更新函数（一般成对获取它们），从而达到 Class 组件 state，setState 相似的目的

React 保持对当先渲染中的组件的追踪，使得在不同组件中调用 useState() 会得到各自独立的本地 state，从而达到保存当前组件状态的目的

2、Effect

  useEffect() 方法可以在函数组件中执行副作用操作，与 Class 组件生命周期函数相似的目的
  常见的副作用如网络请求、手动更新视图，打印日志等（无需清除的副作用）以及数据订阅、计时器等（需要清除的副作用）
  每个 useEffect() 方法中都可以返回一个可选的清除函数（React 将会在执行清除操作时调用它），这是 Effect 的清除机制，从而可以将添加和移除订阅的逻辑放在一起
  Hook 允许我们按照代码的用途分离他们， 而不是像生命周期函数那样。React 将按照 Effect 声明的顺序依次调用组件中的每一个 Effect

  如果某些特定值在两次重渲染之间没有发生变化，不需要执行相应的副作用操作，可以通知 React 跳过对 effect 的调用。传递数组作为 useEffect 的第二个可选参数，其中数组的值为对应状态，只有当状态发生变化的时才会执行 Effect 的调用操作（组件挂载和卸载时执行也会执行）

  特别的，第二个参数传入空数组([])，可以使 Effect 仅在组件挂载和卸载时执行，不依赖于 props 或 state 中的任何值，props 或 state发生改变后也不会重复执行

3、Context

  useContext 能在函数组件中接收 Context 数据

接收一个 context 对象（React.createContext 的返回值）并返回 context 的值（由上层组件中距离当前组件最近的 <MyContext.Provider>提供）。相当于 Class 组件中的 static contextType = Context 或者 <Context.Consumer>组件

代码由 Class 组件中 Context 的示例改写，展示了函数组件与 Class 组件混用的情况。更多的 Context 相关用法见：《React组件与基本语法》中对 Context 的描述

4、自定义Hook

  自定义 Hook，可以将组件逻辑提取到可重用的函数中
  自定义 Hook 是一种自然遵循 Hook 设计的约定，而并不是 React 的特性
  Hook 必须以 “use” 开头，否则 React 将无法自动检查自定义 Hook 是否符合 Hook 规则
  在两个组件中使用相同的 Hook 不会共享 state 状态，从 React 的角度来看，我们的调用 Hook（本身就是函数）只是调用了 useState 和 useEffect 等，影响是完全独立的

  一个自定义Hook定义和使用的例子：
  ① 自定义一个 Hook 名为 useIsPass，返回传入的分数是否及格

  ② 定义分数输入框函数组件 Input，传入初始分数，可编辑对应分数（通过父组件传入的更新函数对父组件对应状态进行更新），引入自定义的 钩子函数 useIsPass 并使用，通过分数及格情况展示不同背景颜色，及格时为绿色，不及格时为红色

  ③ 定义分数提示框函数组件 Label，传入初始分数，引入自定义的 钩子函数 useIsPass 并使用，通过分数是否及格情况展示对应文案

  ④ 定义分数展示组件 ScoreBox, 定义对应分数已经更新函数，引入 Input、Label 组件展示分数情况

  示例效果：

  示例中，我们自定义 useIsPass Hook，在 Input、Label React组件中使用，将根据分数判断是否及格这个逻辑提取到可重用的 useIsPass Hook 函数中，实际自定义 Hook 函数也实现更复杂的场景，如请求接口数据，实现订阅等。Hook 解决了以前在 React 组件中无法灵活共享逻辑的问题
  [注] 无法在 Class 组件中使用函数组件的 Hook，但两者可以在组件树中混用。目前 Hook 还没有 Class 组件一些不常用生命周期的等价写法，React 团队正在迭代，推荐使用函数组件，推荐使用 Hook。推荐把 state 切分成多个 state 变量，使用多个 useState，每个变量包含的不同值会在同时发生变化
